# life-plan-lab

FIRE・住宅ローン・教育費・老後を統合した生活設計シミュレーター（V2）。

「未来を当てる」ためではなく「未来を整理する」ためのツール。入力して終わりではなく、
**結果を見る → 条件を変える → また結果を見る → 家族で話す** という試行錯誤を支えることを重視しています。

## 開発

```bash
npm install
npm run dev      # 開発サーバ（http://localhost:5173）
npm run build    # 型チェック + 本番ビルド（dist/）
npm test         # ロジック/ストア/描画のテスト（vitest）
```

静的アプリとしてビルドし、将来 `/lifeplan-v2/` などのサブディレクトリに独立ページとして
公開する想定（`vite.config.ts` の `base: './'` で相対パス出力）。

## 実機・スマホでの確認手順

1. `npm install`
2. `npm run dev`
3. 表示された URL（既定 `http://localhost:5173`）をブラウザで開く
4. Chrome DevTools のデバイスツールバー、またはスマホ実機で **幅375px（iPhone相当）** にする
5. ざっくり診断を最初から最後まで操作する
6. 結果画面と各 Bottom Sheet を開いて確認する

**スマホ実機（同じ Wi-Fi）で見る場合：**

```bash
npm run dev -- --host 0.0.0.0
```

起動後に表示される **Network URL**（例 `http://192.168.x.x:5173`）をスマホのブラウザで開く。

**開発用ショートカット（`npm run dev` 時のみ表示）：** モード選択画面の下に次のリンクがあります。

- 「サンプルで結果を見る」… ざっくり診断のサンプルで結果画面へ
- 「しっかり診断をサンプル入力で開始」… しっかり診断にサンプルを流し込んで入力フローから確認
- 「しっかり診断サンプルで結果を見る」… しっかり診断のサンプルで結果画面へ
- 「高収入ケースで結果を見る」… 検証用ケース（38歳/年収1200万/資産3200万/子2人/住宅ローン/55歳サイドFIRE）

いずれも本番では非表示。source は `recommended_value` 扱いで、実ユーザーのフローとは分離しています。

## アーキテクチャ

- `src/schema` … 型・入力スキーマ（`Field<T>{value, source, label, assumptionText}`）・標準値/おすすめ値・正規化（両モードの合流点 `buildFullInput`）・ざっくり診断の質問定義
- `src/engine` … 純粋関数の計算群（年次/教育費/住宅ローン/FIRE/判定）。UIから完全分離。定数は `engine/constants.ts`
- `src/store` … zustand。ステップ遷移・入力状態・自動保存（localStorage）
- `src/features` … 画面（mode-select / input-steps / results / resume）

- **今回のポイント**：最大3つで分かりやすい

- 結果画面の結論がすぐ分かるか／Hero が大きすぎないか

### しっかり診断チェックリスト（STEP4 / 4.5）

入口：モード選択で「しっかり診断」、または結果画面の「しっかり診断で詳しく見る」（ざっくりの入力を引き継ぎ）。
開発時は上記の開発用ショートカットでサンプル確認できます。

**カテゴリごと**

- **detailed-basic（基本情報）**：本人/配偶者年齢・世帯/手取り年収・現在資産・現金比率
- **detailed-income（収入）**：本人/配偶者収入・昇給率・退職予定年齢・退職金
- **detailed-expense（支出）**：毎月生活費・年間特別費・車/旅行/保険
- **detailed-family（子ども・教育）**：人数＋子ごとの折りたたみカード（未定はおすすめ値）
- **detailed-housing（住まい・住宅ローン）**：持ち家系のみローン詳細（残高/返済/金利/固定/返済方式/ボーナス）
- **detailed-fire（FIRE条件）**：完全/サイド・希望年齢・FIRE後生活費/収入・就労年齢
- **detailed-investment（投資）**：毎月投資額・想定利回り・インフレ率・暴落シナリオ
- **detailed-retirement（老後）**：年金・老後生活費・医療介護予備費・退職金
- **detailed-events（ライフイベント）**：車/リフォーム/親支援/相続/その他（含める＋年齢＋金額）

**確認観点**

- 1ページが長すぎないか／無駄なスクロールが多くないか／下部固定ナビが邪魔でないか
- Skip / Recommended / Help が圧迫していないか／任意項目が多すぎて疲れないか
- 「未定」「スキップ」が自然に選べるか
- 子ども複数人でも画面が重すぎないか（折りたたみ）／住宅ローン詳細が入力地獄になっていないか
- FIRE条件が完全FIRE/サイドFIREで自然に切り替わるか（サイドのみFIRE後収入・就労年齢が出る）
- 老後・年金・ライフイベントが専門的/複雑すぎないか（入力しなくても進める雰囲気か）
- 途中で戻っても入力値と source が維持されるか／結果画面へ自然に進めるか
- 結果の「条件を変えてみる」から9カテゴリへ戻って再計算できるか

**捕捉のみ項目について（重要）**

配偶者年齢・昇給率・毎月投資額・住宅ローン残高/金利/ボーナス払い・現金比率・暴落シナリオなどは、
現在は「入力内容として保持」しつつ、試算には反映していません（精密反映は次STEP）。
結果画面の「今回の試算条件」には、その旨の注記を表示しています（入力したのに結果が変わらない不信感を避けるため）。

## 計算の反映ステータス（STEP5）

コード上は `src/engine/annualSimulationEngine.ts` 冒頭のコメントに、表示上は結果画面の「今回の試算条件」注記に反映状況を記載しています。

**反映済み**：年齢／**手取り年収（直接入力＞夫婦別収入の手取り換算合算＞世帯年収×年収帯手取り率）**／
**昇給率（現役期に年次反映）**／退職予定年齢（以降は労働収入0）／現在資産（**現金/投資に分割**）／
**現金比率（投資資産のみ利回り）**／**毎月投資額（現金→投資への振替。収支に二重加算しない）**／毎月生活費／
住宅費（賃貸=家賃・持ち家=ローン返済を完済年齢まで＋**完済後維持費 年60万**）／子ども別年齢→教育費／教育方針／
想定利回り（名目）／インフレ率（支出側）／FIRE種別・希望年齢／FIRE後生活費／サイドFIRE後収入・就労終了年齢／
年間特別費・車・旅行・保険／退職金（**FIRE/退職年に一括**）／年金（65歳〜・入力時）／医療介護予備費（75/85歳〜）／
ライフイベント支出・収入（相続は収入・断定しない表現）

**簡略反映**：持ち家維持費は定額（年60万）／教育費は初期値テーブル／税（0）／手取り率は年収帯ざっくり

**未反映（次STEP）**：住宅ローン残高・金利・固定変動・返済方式・ボーナス払いからの償却／配偶者年齢／
NISA・iDeCo・各種控除／所得税・住民税・社会保険料の精密計算／暴落シナリオ／夫婦別の退職年齢・年金

**資産モデル（重要）**：資産を現金資産と投資資産に分け、利回りは投資資産にのみ適用。年間収支は現金へ加え、
現役期かつ黒字のときのみ毎月投資額（未入力時は黒字の一部）を現金→投資へ移動（赤字年は新規投資を停止）。
赤字は現金→投資の順で取り崩し。毎月投資額は「新規投入額」であり収支に二重加算しません。

**運用順序（保守的）**：利回りは「年初の投資資産」にのみ適用し、その年の新規投入は翌年から運用します。

**枯渇の扱い（STEP5.1で修正）**：現金・投資資産はマイナスにしません。取り崩せない不足分は
`cumulativeShortfall`（累計不足額）に蓄積し、**マイナス残高に負の利回りをかけない**（負の複利を発生させない）。
表示資産は0でクランプし、結果には「◯歳で資産が尽きる見込み・95歳時点の累計不足額：約◯◯万円」と表示します。

**FIRE準備率（4%ルール）は参考指標**：Hero では資産寿命・95歳残資産を主に見せ、FIRE準備率は目安として併記。
年金未入力時はその旨を注記し、年次シミュレーションの資産寿命とあわせて確認するよう案内します。

> ざっくり診断にも、現実感のための最小項目（毎月生活費・毎月住居費・住宅ローン残年数・子の年齢・FIRE後生活費・サイドFIRE後収入）を追加しました。
> これらは同じ意味のしっかり診断項目へ引き継がれ、別管理になりません（buildFullInput で合流）。
